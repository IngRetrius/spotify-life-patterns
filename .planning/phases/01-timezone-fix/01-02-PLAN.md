<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Local dev -> Supabase (production DB) | psycopg2 pooler connection writes to sessions / session_features / activity_labels |
| Dashboard -> Supabase | SQLAlchemy read-only queries over the same pooler |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | Destructive update corrupts session_id or start_time | mitigate | `DO UPDATE SET` clause restricts mutation to `hour_of_day` and `day_of_week` ONLY (see UPSERT_SESSION_SQL lines 138-140). session_id and start_time are untouched, preserving foreign keys from session_features and activity_labels. |
| T-02-02 | Information Disclosure | Credentials printed in stdout during re-run | accept | `scripts/run_pipeline.py` and `db/connection.py` never print password; password is resolved via `_resolve_password()` from env/secrets and passed directly to psycopg2.connect. No change in this plan. |
| T-02-03 | Denial of Service | Re-run during dashboard read causes transient outage | accept | Single-user dashboard, batch pipeline every 6h. A few seconds of elevated load is acceptable. Upsert is a single transaction committed atomically (`conn.commit()` after all rows). |
| T-02-04 | Repudiation | No record of what was changed | mitigate | SUMMARY.md in this plan captures before/after row counts, verification query output, and re-run timestamp. Log files persist the full stdout. |

No high-severity threats apply. Rationale: mutation scope is tightly bounded by `DO UPDATE SET` to two non-identifying integer columns; FK integrity is guaranteed by session_id stability; credentials never leave `_resolve_password`.
</threat_model>

Exact command from repo root:

```bash
python - <<'PY' | tee .planning/phases/01-timezone-fix/01-02-verification.txt
from db.connection import get_connection

Q_DIAGNOSTIC = """
    SELECT session_id, hour_of_day,
           EXTRACT(HOUR FROM start_time AT TIME ZONE 'America/Bogota')::int AS expected_hour,
           hour_of_day - EXTRACT(HOUR FROM start_time AT TIME ZONE 'America/Bogota')::int AS diff
    FROM sessions
    WHERE hour_of_day != EXTRACT(HOUR FROM start_time AT TIME ZONE 'America/Bogota')::int
    LIMIT 20;
"""

Q_MISMATCH_COUNT = """
    SELECT COUNT(*) AS mismatched_rows
    FROM sessions
    WHERE hour_of_day != EXTRACT(HOUR FROM start_time AT TIME ZONE 'America/Bogota')::int;
"""

Q_TOTAL = "SELECT COUNT(*) AS total_rows FROM sessions;"

Q_DOW_MISMATCH = """
    SELECT COUNT(*) AS dow_mismatched_rows
    FROM sessions
    WHERE day_of_week != EXTRACT(ISODOW FROM start_time AT TIME ZONE 'America/Bogota')::int - 1;
"""

conn = get_connection()
cur = conn.cursor()

print("=== Verification: hour_of_day vs Bogota-local hour ===")
print()

print("--- Q1: Diagnostic sample of mismatched rows (should be EMPTY) ---")
cur.execute(Q_DIAGNOSTIC)
rows = cur.fetchall()
if not rows:
    print("(0 rows)  <-- PASS: no mismatches found")
else:
    print("session_id | hour_of_day | expected_hour | diff")
    for r in rows:
        print(" | ".join(str(x) for x in r))
    print("  ...  showing up to 20 of potentially more mismatched rows")
print()

print("--- Q2: Count of mismatched hour_of_day rows ---")
cur.execute(Q_MISMATCH_COUNT)
(mismatch,) = cur.fetchone()
print(f"mismatched_rows = {mismatch}")
print()

print("--- Q3: Total rows in sessions (sanity check) ---")
cur.execute(Q_TOTAL)
(total,) = cur.fetchone()
print(f"total_rows = {total}")
print()

print("--- Q4: Count of mismatched day_of_week rows (Mon=0 .. Sun=6) ---")
cur.execute(Q_DOW_MISMATCH)
(dow_mismatch,) = cur.fetchone()
print(f"dow_mismatched_rows = {dow_mismatch}")
print()

print("=== VERDICT ===")
if mismatch == 0 and dow_mismatch == 0:
    print(f"PASS: {total} rows, 0 hour mismatches, 0 day_of_week mismatches.")
else:
    print(f"FAIL: {mismatch} hour mismatches, {dow_mismatch} dow mismatches out of {total} rows.")

cur.close()
conn.close()
PY
```
