<objective>
Reduce the dashboard's per-load payload by paginating the Sessions detail table server-side and by replacing the pandas-based day filter in the Day Detail section with a dedicated SQL query.

Purpose: `load_sessions` currently pulls every row of `sessions` joined with features and labels on every cache miss; with thousands of rows and a hibernating Streamlit Cloud app this is wasteful and slow. The Day Detail section compounds the problem by re-downloading the full table just to filter one calendar day in pandas.

Output: Updated `dashboard/queries.py` with paginated + day-scoped loaders, and updated `dashboard/app.py` with a page control for the Sessions table and a day-scoped loader for Day Detail.
</objective>
